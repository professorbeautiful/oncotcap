Statement Template editor window


PURPOSE:
  To connect an ordinary language statement representing a modeling idea to an agent-based representation of that idea, so that the idea can be implemented in a working model simulation.

HOW TO USE THIS WINDOW:
 

You can obtain this window in several ways:

  • If is the root of the browser tree, click .
  • Whenever is clicked to create a new StatementBundle, and the window "Add a StatementBundle" appears, at the bottom clikc on a button labeled "create SB from new ST".

The Statement Template editor is divided into three areas:

  • Upper left: template text area.
  • Upper right: parameter table.
  • Bottom: list of Code Bundles and Statement Bundles which implement this Statement Template.

Type into the template text area. To insert a parameter into the text, type alt-A or click the word add (upper right). See also:

  • Creating and Editing Parameters

Add Code Bundles and Statement Bundles to the bottom table in order to allow this template to "come to life" in a simulation. See also:

  • Empowering a Statement Template with Code Bundles and Statement Bundles

TIPS:
  Note that the arrow keys jump over parameter words.

KNOWN BUGS & INCONVENIENCES:
   When adding a parameter, new lines are inserted into the text. These can be removed individually with the delete key. The cause is a basic bug in Java's HTML editor kit.
