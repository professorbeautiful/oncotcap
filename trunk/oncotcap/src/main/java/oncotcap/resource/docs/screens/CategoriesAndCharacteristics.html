Parameters involving Categories and Characteristics

Categories

A Category is a Keyword chosen from a sub-tree of the Keyword tree.
A Category parameter is configured by naming it and providing the root node of the sub-tree from which the value should be chosen.
A Category, then, corresponds roughly to a Concept in the NCI MetaThesaurus.

Example:
In the Statement Template 

"Cells of type cell-type respond to agent by setting the mechanism to new state".

the parameter cell-type is a Category parameter. Its "parameter creation editor" allows one to set the name, here cell-type, and to set the root node to the Keyword "Cell type".

When this StatementTemplate is used in a Statement Bundle, and the Statement Bundle user clicks on the hypertext-style parameter name to open its "parameter value editor", the parameter's value can be assigned to any Keyword that is in the subtree of the Keyword "Cell type" node. Then, just as with all parameters, the hypertext parameter name is replaced by the selected value. For example the statemenet Bundle may end up looking like this:

 "Cells of type cancer cell respond to vincristine by setting the efflux pump mechanism to on."

 

Characteristics

A Characteristic is a parameter whose allowable values are taken from a small set.
A Characteristic corresponds roughly to a "common data element" (CDE) which is denumerable, i.e. an Enumeration.
The allowable values are called "levels". They are stored in "level lists".
Each level list is connected to a Keyword in the Keyword tree.
